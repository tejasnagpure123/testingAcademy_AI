3. Request/Response Formats & Authentication





JSON: All requests and responses are expected to be in JSON format.



Content-Type Header: Content-Type header must be set to application/json for POST, PUT, and DELETE requests.



Authentication:





Cookie Authentication: Uses a token cookie for authentication.



Basic Authentication: Uses Basic Authentication (username/password) – requires the Authorization header to be set: Authorization: Basic <base64 encoded username:password>.

4. Endpoint Details





Partial Update (PUT): The PUT endpoint allows you to update only the fields you provide in the request body. It doesn’t require you to send all fields.



Delete (DELETE): The DELETE endpoint requires authentication to prevent unauthorized deletion.

5. Response Codes





200 OK: Success for GET requests and successful POST/PUT/DELETE operations.



201 Created: Success for the POST operation (creating a new booking).



201 Created Health Check endpoint.
