# Image Prompts

## Instagram Carousel Image Prompt

Create a 9-slide Instagram carousel, 4:5 ratio, 1080x1350 per slide, consistent clean infographic style.

Style: Clean Infographic, dark. Background deep navy #0a0a1a with subtle grid texture. Headers in gold/amber #f59e0b. Warnings and unsafe actions in red #ef4444. Safe actions and verification checks in green #22c55e. Body text in white. Use JetBrains Mono for code and numbers. Use bold Inter-style sans-serif for labels. Flat vector design, no 3D, no glossy icons. Mobile-readable at 30%. Small muted gray @thetestingacademy watermark bottom-right on every slide.

Slide direction:

Slide 1: Big hook text: "Your AI agent needs a QA contract." Subtext: "Not a longer prompt." Minimal contract document icon.

Slide 2: Split visual. Left: messy long prompt stack in red. Right: clean contract checklist in green. Title: "Prompt debt hides risk."

Slide 3: Four-column ICSR framework: Instructions, Context, Skills, Rules. Use simple icons: document, folder, wrench, shield.

Slide 4: Before/after instruction example. Red card: "Create tests." Green card: "Create one Playwright spec from Jira. Do not edit app code."

Slide 5: Allowed context map. Show Jira, existing tests, Playwright config, helper docs flowing into the agent. Everything else blocked by a red boundary.

Slide 6: Skills permission grid. Green: read files, write one spec, run targeted test. Red: push commits, edit CI, invent credentials.

Slide 7: Code-focused slide with large readable snippet:
`getByRole("button", { name: /apply coupon/i })`
`expect(getByRole("status")).toContainText("Coupon applied")`
Caption: "No user-visible assertion? No trust."

Slide 8: Output schema checklist: files changed, assumptions, test cases covered, commands run, risks, human review notes.

Slide 9: Final thesis card. Big text: "Prompt tells the agent what to do. Contract tells the team what must never break." CTA: "Comment CONTRACT."

Important: Keep text accurate and readable. If text generation fails, recreate the carousel in Figma or Canva using this layout.
